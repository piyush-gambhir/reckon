A Claude Code workspace that wires Grafana, Jenkins, and CubeAPM CLIs together under isolated, folder-scoped credentials — so a coding agent can correlate signals across systems in a single session.

MIT License Claude Code workspace direnv-based isolation

Prerequisites

RCA Assist composes three CLIs and uses direnv for credential isolation. Install them once, globally.

Tools
direnvFolder-scoped environment variables — direnv.net
grafana-cliDashboards, datasources, alerts, annotations
jenkins-cliJobs, builds, pipelines, logs, nodes
cubeapm-cliDistributed traces, PromQL metrics, LogsQL logs
Install direnv
brew install direnv           # macOS
sudo apt install direnv       # Ubuntu / Debian

# Then add the hook to your shell rc file
eval "$(direnv hook zsh)"     # or bash, fish
Install the three CLIs
go install github.com/piyush-gambhir/grafana-cli@latest
go install github.com/piyush-gambhir/jenkins-cli@latest
go install github.com/piyush-gambhir/cubeapm-cli@latest

Setup

Clone the workspace, allow direnv, authenticate each CLI, and seed your team-specific knowledge files.

1. Clone the workspace
git clone https://github.com/piyush-gambhir/rca-assist.git
cd rca-assist
2. Allow direnv
direnv allow

This loads the isolated credential environment defined in .envrc.

3. Authenticate each tool
grafana login
jenkins login
cubeapm login

Credentials land in .config/ inside this directory — never in your global ~/.config/.

4. Seed team knowledge files
cd infra-knowledge
for f in *.example.md; do cp -n "$f" "${f/.example/}"; done
cd ..

Then edit each infra-knowledge/<topic>.md with your real service names, on-call contacts, runbooks, etc. They're gitignored — your data stays local.

Verify

Confirm all three connections are live before opening the workspace in Claude Code.

grafana user current -o json
jenkins status -o json
cubeapm traces services -o json
How credentials are isolated. .envrc sets XDG_CONFIG_HOME to .config/ inside this directory. Each CLI reads its config from there, fully isolated from your global ~/.config/ profiles. Each clone of this repo can hold completely different credentials — useful for staging / prod / customer environments.

Workflow

Open the folder in Claude Code and ask the agent to investigate.

Open in Claude Code
cd rca-assist
claude
Example prompts
Investigate"Investigate the apdex drop on media-service over the last 30 min."
Correlate"Correlate the latest deploy of api-gateway with error spikes."
Bisect"Find the Jenkins build that introduced the latency regression."
Summarize"Summarize alerts firing right now and their likely root cause."
Document"Write an RCA doc for incident X using traces, metrics, and build history."
