## What it is

`rca-assist` is an agent workspace that turns a coding agent into a disciplined on-call engineer. When an alert fires, you hand the agent the alert text and it runs a cascade — symptom → service → endpoint → sub-span → downstream → trigger — and writes a structured RCA at the end. The spine is three CLIs (`grafana`, `jenkins`, `cubeapm`); the workspace also wires `aws`, `gh`, `kcat`, `rpk`, `kubectl`, `redis-cli`, `mongosh`, `psql`, `mysql`, and optionally `es` (all read-only) for cross-checks when APM signals are thin. It runs under Claude Code, Codex CLI, or any runtime that reads `AGENTS.md` — with subscription login or API keys.

It is **production-only** by design: each clone holds production credentials in `.env` and isolates them via `direnv`. If you need staging or UAT, use a separate clone so the agent never crosses environments mid-investigation.

## How a typical session looks

1. Alert fires in your channel: *`[#NNNNN] [FIRING:1] Payments API | P2-Warning: Error Rate >3% (10min)`*.
2. You paste the alert into the agent. It starts the cascade automatically.
3. Agent identifies the failing endpoint, the upstream cause (e.g., a third-party API rejection), confirms novelty over 30 days, and pinpoints recovery time.
4. Agent writes the RCA + raw evidence into `incidents/<UTC-date>-<slug>/`. You review and ship it.
5. Agent runs a post-incident self-review and routes any learnings into the skill or `infra-knowledge/` so the next on-call won't repeat the same mistakes.

## Built on

The cascade spine:

- **[CubeAPM CLI](https://github.com/piyush-gambhir/cubeapm-cli)** — distributed traces, PromQL metrics, LogsQL logs.
- **[Grafana CLI](https://github.com/piyush-gambhir/grafana-cli)** — dashboards, datasources, alerts, annotations.
- **[Jenkins CLI](https://github.com/piyush-gambhir/jenkins-cli)** — jobs, builds, pipelines, logs, nodes.

Plus a read-only ops toolbelt for cross-checks: **aws** (CloudWatch / ALB / SQS / ECS), **gh** (PRs, Actions, releases), **kcat** + **rpk** (Kafka), **kubectl** (pod/deploy state), **redis-cli** (cache diagnostics), **mongosh** / **psql** / **mysql** (read-only DB shells), and optionally **[es](https://github.com/piyush-gambhir/es-cli)** (Elasticsearch/ELK log stores, client-enforced read-only). A local **incidents viewer** (`node ui/serve.mjs`) renders your RCA corpus in the browser. And the runtime:

- **[Claude Code](https://docs.anthropic.com/en/docs/claude-code)** — reads `CLAUDE.md` and loads the skill. Subscription or `ANTHROPIC_API_KEY`.
- **[Codex CLI](https://github.com/openai/codex)** / **[OpenCode](https://opencode.ai)** — read `AGENTS.md`. ChatGPT subscription or `OPENAI_API_KEY`.
