## What it is

`rca-assist` is a Claude Code workspace that turns a coding agent into a disciplined on-call engineer. When an alert fires, you hand the agent the alert text and it runs a cascade — symptom → service → endpoint → sub-span → downstream → trigger — using three CLI tools (`grafana`, `jenkins`, `cubeapm`) and writes a structured RCA at the end.

It is **production-only** by design: each clone holds production credentials in `.env` and isolates them via `direnv`. If you need staging or UAT, use a separate clone so the agent never crosses environments mid-investigation.

## How a typical session looks

1. Alert fires in your channel: *`[#36517] [FIRING:1] Conversational AI | P2-Warning: Error Rate >3% (10min)`*.
2. You paste the alert into the agent. It starts the cascade automatically.
3. Agent identifies the failing endpoint, the upstream cause (e.g., a Twilio API rejection), confirms novelty over 30 days, and pinpoints recovery time.
4. Agent writes the RCA + raw evidence into `incidents/<UTC-date>-<slug>/`. You review and ship it.
5. Agent runs a post-incident self-review and routes any learnings into the skill or `infra-knowledge/` so the next on-call won't repeat the same mistakes.

- **[Grafana CLI](https://github.com/piyush-gambhir/grafana-cli)** — dashboards, datasources, alerts, annotations.
- **[Jenkins CLI](https://github.com/piyush-gambhir/jenkins-cli)** — jobs, builds, pipelines, logs, nodes.
- **[Claude Code](https://docs.anthropic.com/en/docs/claude-code)** — the agent runtime and skill loader.
